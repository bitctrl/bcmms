Axis Communications devices such as IP cameras are supported as input sources for video streams by bitcontrol® Digital TV Link via the plug-in AXIS.

The plug-in can be started in the main window (see Figure 12.1, “Plug-in AXIS - Main window”) with the help of the the Start button and can be stopped with the Stop button.

Figure 12.1. Plug-in AXIS - Main window


12.1. About

The window About shows information (e.g. license, license key, release level, ... ) about the plug-in AXIS.

Figure 12.2. Plug-in AXIS - About


12.2. Camera

The window Camera contains a survey of all available AXIS-cameras (in the connected network) and their properties (see Figure 12.3, “Plug-in AXIS - AXIS camera with different codecs”.

Note

It is possible that the overview is still empty after the first call. In this case a camera search with the Search Camera button can be executed. Another possibility would be camera addition by hand (see Section 12.2.1, “Add camera manually”).

The properties can be edited. For this the user clicks the right mouse button on the respective field and changes the entry.

Figure 12.4, “Example: AXIS camera video with time and logo” shows as an example the video of the AXIS camera from the first line of Figure 12.3, “Plug-in AXIS - AXIS camera with different codecs”. The time and the AXIS logo are blended in the video just as they are set as properties in the table fields.

Figure 12.4. Example: AXIS camera video with time and logo


Note

All properties will be delivered by the AXIS camera itself. The bitcontrol® Digital TV Link table is only used to adjust the preset properties. The entries must be assumed afterwards with the Apply button. They will be used after switching.

12.2.1. Add camera manually

To add an AXIS camera manually, the right mouse button will be used in the first step. The submenu shown in Figure 12.5, “Submenu in AXIS camera overview” appears. After selecting the entry Add Camera... a new entry for an AXIS camera appears in red color in the list (see Figure 12.6, “New AXIS camera entry”). The properties can now be edited. After the affirmation of the settings with the Apply button, the created camera entry changes its color from red to black (see Figure 12.7, “Assumed AXIS camera ”).

Figure 12.5. Submenu in AXIS camera overview


Figure 12.6. New AXIS camera entry


Figure 12.7. Assumed AXIS camera


12.2.2. Delete/Remove camera

To delete/remove an AXIS camera, the right mouse button will be used in the first step. The submenu shown in Figure 12.5, “Submenu in AXIS camera overview” appears. After selecting the entry Remove..., the selected camera will be deleted from the list.

12.3. Statistics

The statistic information of the AXIS plug-in is self-explanatory in its designations.

The values are updated in real-time, so the information about AXIS video stream decoding is always up to date. If the value of Renderer Lateness grows permanently, the hardware (CPU, graphic chip, bus system) will be overloaded. In this case, it is very advisable to try using DXVA.
