The tables of the windows Video Codec Mapping (see Figure 7.1, “Video Codec Mapping table”) and Audio Codec Mapping contain codec entries which can be used for bitcontrol® Digital TV Link in addition to the one which is set in the Preferences window. The codec which is set in the Preferences window will be used primarily followed by the codec in the table in descending order.

Note

The settings will be explained using the example Video Codec Mapping. Audio Codec Mapping works analogously.

Figure 7.1. Video Codec Mapping table


ForCCs: This table column contains the video formats to which a video codec will be allocated. The entry can be edited by double clicking on the entry.

Codec: This table column contains the codec that is allocated to the FourCCs. After double clicking on an entry, the selection field Codec (see Figure 7.3, “Add mapping entry”) appears. The selection field shows all codecs which are installed on the system and the codecs which were created individually (see Chapter 8, Decoder / Encoder / Transcoder). The codec will be selected from these possibilities.

Note

The selection list also contains DMO (Direct Media Object) codecs. These codecs shouldn't be used in bitcontrol® Digital TV Link version 3.8. A respective wrapper is in preparation.

7.1. Add mapping entry

By right clicking in the table a context menu (see Figure 7.2, “Mapping context menu”) opens. The entry Add will be selected here.

Figure 7.2. Mapping context menu


A row will be added to the table. FourCC (video format) and the respective codec will be entered here.

Figure 7.3. Add mapping entry


7.2. Remove mapping entry

The context menu entry Remove is used to remove the marked codec from the table.

Note

There will be no additional dialogue for the confirmation of codec deletion.

7.3. Change priority of a mapping entry

The mapping context menu (see Figure 7.2, “Mapping context menu”) contains the entries Move up and Move down which can be used to move a mapping entry one position up or one position down, so the priority will change too. Entries in the upper part of the table have a higher priority than the entries in the lower part of the table.
