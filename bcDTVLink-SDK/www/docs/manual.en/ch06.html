6.3. Plug-ins (in Channels window)

Everything that was parameterized in the Encoder-, Decoder- and Transcoder windows will be used in the Channels windows, because the allocation of Encoders, Decoders and/or Transcoders to the channels of plug-ins occurs here. Furthermore, it is possible to switch the Channels or data streams as well as to provide them to the Multistream Server plug-in here.

6.1. Active window

The Active window (see Figure 6.1, “Active window of the Channels dialogues”) is opened by clicking on the tree entry Channels in the left part of the Options dialogues.

Figure 6.1. Active window of the Channels dialogues


The Active window is separated in two parts: The upper part contains a table which shows all active channels and/or data streams of plug-ins that were activated by the Start button in the respective configuration window (see Section 6.3, “Plug-ins (in Channels window)”), and their properties. The lower table shows the data streams of the Channels marked in the upper table and their properties.

The following Channel properties can be found in the columns of the upper table:

ID: ID of the Channel

Channel: Name of the Channel

Source: Shows the respective plug-in as the Channel source

Primary: Shows which plug-in Channel is running at the moment

OnAir: Shows if a Channels data stream is transmitted to the Multistream Server plug-in (on) or not (off). The setting here corresponds with the OnAir setting of the lower table. If OnAir is deactivated here (off), the property will not be available anymore in the lower table. The fields will be greyed out there.

Record: Shows if the Channel is being recorded at the moment (If so, the field contains the marker on. As long as the record is initiated the field contains the marker Starting....)

VideoTracks: Shows how many video streams are included in the Channel

AudioTracks: Shows how many audio streams are included in the Channel

Publisher: Shows if a Channel is published at the moment. (If so, the field contains the marker on. As long as the publishing is initiated the field contains the marker Starting....)

Provider: Shows the Channels provider

EPG: Shows further Channel information (That could be for example program information from EPG or own notes, e.g. camera location)

The following buttons are available under the upper table:

Switch: The button Switch is used to change the Channel

OnAir: This button sets the OnAir flag for the marked Channels to on or to off.

Publish: This button starts or stops the Channel publishing in network.

Record: This button is used to start or stop the Channels record.

Note

The functionalities of the button are also available in the Channels context menus (see Figure 6.2, “Context menu of plug-in Channels”).

The columns of the lower table show the following properties of the data streams of the selected Channels:

ID: ID of the data stream (This ID can be used as SubstreamID in the respective table field in the Autostart window, to address the data stream for an Autostart action. The conversion of the ID occurs automatically there.)

FourCC: Shows the data stream format

Type: Shows the type of data stream

Language: Language of the data stream

Primary: Shows which data stream of a type is active at the moment

OnAir: Shows if this data stream is provided to the Multistream Server plug-in

Description: Shows more detailed information about the data stream

UpTime: Shows how long the data stream has already been running

The following buttons are available under the lower table:

Switch: This button is used to switch between the data stream of the same type.

OnAir: This button sets the OnAir Flag for the marked data stream to on or to off.

Transcode: This button is used to change the Decoder, Encoder and Transoder settings of the marked data stream (see Figure 6.4, “Submenu to change Transcoder settings”, Figure 6.5, “Submenu to change Decoder settings” and Figure 6.6, “Submenu to change Encoder settings”).

Note

The functionalities of the button are also available in the Channels context menus (see Figure 6.3, “Data stream context menu”).

6.2. Actions window

The Actions window (see Figure 6.7, “Actions window” ) contains rules for a Channel, which will be executed consecutively during the startup of bitcontrol® Digital TV Link.

Figure 6.7. Actions window


The following settings for Actions functions can be set in the columns of Actions table:

Action: Name of the Actions function

Parameters: Parameter of the Actions function

Source: Source of the data stream

Channel: Channel of data stream

StreamID: ID of the data stream

SubstreamID: ID of the substream of the data stream (see column ID in the lower table of Active window, Section 6.1, “Active window”)

SubstreamType: Type of the substream

SubstreamFCC: Data format of the substream

Delay (sec): Time of delay in seconds

Table entries can be added (Add...), removed (Remove...) or changed in priority (Move Up and/or Move Down) with the Actions context menu (see Figure 6.8, “Actions context menu” ).

Figure 6.8. Actions context menu


6.3. Plug-ins (in Channels window)

Sliders for all started plug-ins are available in the Channels window too, beside the sliders Active and Autostart.

In the respective plug-in window (examples see Figure 6.9, “Example Channel window of Dreambox plug-in” and Figure 6.10, “Example Channel window of AXIS plug-in”) all available channels of the plug-in will be shown in a table.

Figure 6.9. Example Channel window of Dreambox plug-in


Figure 6.10. Example Channel window of AXIS plug-in


All plug-in channels have the following properties:

Name: Name of the channel

Active: Shows if the channel is active at the moment (yes) or not (no)

The channel of a plug-in can have further properties (like URL, provider, ...). These depend on the respective plug-in.

Channels can be started or stopped via the context menu (see Figure 6.11, “Channel plug-in context menu”). The context menu is opened by right clicking on a Channel entry.

Figure 6.11. Channel plug-in context menu


The Start button is used to switch Channels inside the plug-in.

The started and/or switched Channel will be added afterwards to the table in the Active window (siehe Section 6.1, “Active window”).

Note

A Channel can provide several data streams. Therefore it is possible that a Channel only respresents one of these data streams. If so, the Start button switches the data streams.
