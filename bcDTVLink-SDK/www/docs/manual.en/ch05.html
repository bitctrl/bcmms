5.10. Window Plugins

The bitcontrol® Digital TV Link has several configuration windows for calling up information and configuring the application. It is possible to open the bitcontrol® Digital TV Link configuration and properties dialogues via the bitcontrol® Digital TV Link sytray icon in the footline of the desktop.

Note

The bitcontrol® Digital TV Link provides tooltips. These appear when the user hovers the cursor over an entry.

5.1. Window Options

After Windows Media® Player starts via desktop icon or task bar , the window Options (Figure 5.1, “Window Options”) opens.

Note

The available plug-in in tree depends on the loaded plug-ins. Only loaded plug-ins are visible.

Figure 5.1. Window Options


5.1.1. Editing plug-in entries

5.1.1.1. State change

If a plug-in status should be changed (see Figure 5.1, “Window Options”, right window, second column), the user has to first mark the respective plug-in and then click the right mouse button. The submenu shown in Figure 5.2, “Plug-in context menu” opens. The user selects the respective entry out of this submenu and clicks on it. The change will be done.

The following state changes are possible:

StateDescriptionCondition
LoadActivates the plug-inPlug-in must be in Unload state
StartStarts the plug-inPlug-in must be in Stop state
StopStops the plug-inPlug-in must be in Load state
UnloadUnloads/Deactivates the plug-inPlug-in must be in Stop state

Note

Editing plug-in entries (e.g. status change) is useful if several plug-ins are installed but just some of them are in use. Example: The plug-ins of all supported STB types are installed, but just one of these types is in use.

Figure 5.2. Plug-in context menu


5.1.1.2. Add plug-in

Via the plug-in context menu (see Figure 5.2, “Plug-in context menu”) entry Add... a new plug-in can be added. For this the dialogue shown in Figure 5.3, “Add plug-in” opens.

Figure 5.3. Add plug-in


The name of the new plug-in will be entered into the field Name, the type will be selected out of the selection field Class. The settings can be confirmed by the OK button and the new plug-in appears in the plug-in overview (see Figure 5.4, “New plug-in in the plug-in overview”). If the Cancel button is used, the process will be aborted.

Figure 5.4. New plug-in in the plug-in overview


5.1.1.3. Remove/Delete plug-in

To remove a plug-in from the local instance or to delete it permanently it should be marked in the first step. Afterwards the right mouse button must be used and the submenu opens. The entry Remove... opens the note shown in Figure 5.5, “Remove/Delete plug-in”.

Figure 5.5. Remove/Delete plug-in


The button Remove removes the selected plug-in out of the plug-in overview (local instance).

The button Delete deletes the selected plug-in permanently.

The button Cancel aborts the process.

The window Options is started in controlled operation by clicking on the bitcontrol® Digital TV Link systray icon in the footline with the right mouse button.

Figure 5.6. Start window Options via DTVLink icon

5.3. Window Preferences

With the Preferences window the Audio Output Pin and the Video Output Pin can be activated/deactivated and configured.

Figure 5.8. Window Preferences


Video Output Pin: If this checkbox is active, the video signal will be forwarded from bitcontrol® Digital TV Link to the next DirectShow® Filter. This option is active by default.

Video Decoder: The selection field Video Decoder contains all video decoders that are installed on the Windows® system (see Figure 5.9, “Available video decoder (example)”). The user can select the video decoder that should be used by bitcontrol® Digital TV Link here. Default value: auto

Note

The codecs in the selection field list are third-party software, so we can't warrant for compatibility and operability.

Note

Auto ... Automatic video decoder selection

Default ... Standard video decoder selection

Figure 5.9. Available video decoder (example)


Video Format: The selection field Video Format contains all selectable video formats (see Figure 5.10, “Selectable video formats”). The user can choose the video format that should be used by bitcontrol® Digital TV Link here. It is also possible to edit the list entries and to add new lines.Default value: MPEG2, 720x576, PAL

Syntax: FOURCC - resolution - PAL, NTSC or frequeny (in HZ)

Figure 5.10. Selectable video formats


Audio Output Pin: If this checkbox is active, the audio signal will be forwarded from bitcontrol® Digital TV Link to the next DirectShow® Filter. This option is active by default.

Note

bitcontrol® Digital TV Link selects the audio decoder automatically. Here the user sets the audio format of the audio output pin.

Audio Format: The selection field Audio Format contains all selectable audio formats (see Figure 5.11, “Selectable audio formats”). The user can choose the audio format, which should be used by bcDTVLink here. Default value: MP2, 48000, 16-bit, stereo

Figure 5.11. Selectable audio formats


Buffering Time (msec): Time of buffering in milliseconds

Note

The Buffering Time shoud be individually parametrized for each plug-in, because Buffering Time contains Jitter balance. The buffer should generally be greater than the difference between Jitter Flood and Jitter Famine. Both values can be found in the statistics table when the plug-in is running. Example settings for Buffering Time are: 400ms up to 500ms for plug-in AXIS, 2000ms up to 3000ms for plug-in Dreambox or 1000ms up to 1500ms for plug-in BDA.

Preffered media language: This selection field (see Figure 5.12, “Language selection for plug-ins”) should be used to set the language of the video stream which is delivered by the channel of the selected plug-in.

Figure 5.12. Language selection for plug-ins


Adjust Time Jitter: If this option is active, the video and audio line will be synchonised.

Note

Usually this function isn't necessary. But if the audio streams runs faster than the video stream audio stream fragments must be discarded to ensure lip synchrony, which may cause popping sounds.

Show Console: If this option is active, the Console will be opened automatically after each startup of bitcontrol® Digital TV Link. For more information about this topic see Chapter 11, Console.

Set OnAir by default: If this option is on, the Server plug-in will start automatically during each startup of bitcontrol® Digital TV Link.

Restore last channel: If this option is active, the last channel will be automatically restored during the next startup of bitcontrol® Digital TV Link.

Prefer AC-3 Audio: If this option is active, AC-3 audio format will be preferentially used.

Note

AC-3 requires a respective decoder which must be installed on the system.

Save Log: If this option is active, a bitcontrol® Digital TV Link logfile will be saved. The path to the logfile can be entered in the text field beside the option box or selected by the Browse button. An example logfile is shown in Figure 5.13, “DTVLink logfile (example)”.

Figure 5.13. DTVLink logfile (example)


The first four columns contain internal values which could be useful to find the reasons for errors. Date, Time and event duration will be shown next. The last column has information on which program occurs which action. The last event stands last.

FourCC Map Table: The button FourCC Map Table opens a window (see Figure 5.14, “FourCC Map Table” that contains a list of all bitcontrol® source and target formats. In case of error send the list to the BitCtrl support team. Target FourCC lists all formats supported by bcDTVLink. Source FourCC are formats of video sources. They will be mapped to the respective bcDTVLink Tagret FourCC. The Source FourCC format list can be enhanced by clicking on the desired a format field.

Figure 5.14. FourCC Map Table


The preferred formats are situated on top of the list. But the sequence can be changed with the buttons Move up and/or Move down. For this the desired line has to be selected. A marked line can be removed out of the list with the button Remove.

The button Apply confirms the settings.

The button OK closes the window.

The button Cancel aborts the process and closes the window.

5.3.1. Export of bcDTVLink settings (bcDTVLink scenario)

The bitcontrol® Digital TV Link is able to call up prefabricated configurations. These require the storage of scenarios. With these scenarios it is possible to start bcDTVLink directly with stored settings.

A scenario represents a script which creates, configures and manipulates bitcontrol® Digital TV Link objects. A versed user should be able to control bcDTVLink functions during runtime via this scripting interface (see SDK).

A bcDTVLink scenario will be created with the help of the button Export Settings in the option window Preferences. A characteristic Windows dialogue opens where the storage location of the export/scenario file can be defined (see Figure 5.15, “ Storage location of the export/scenario file”).

Figure 5.15.  Storage location of the export/scenario file


With the Save button the bcDTVLink settings will be combined to an export/scenario file and stored. If the process was successful, the note shown in Figure 5.16, “Export successful” appears.

Figure 5.16. Export successful


The export/scenario file can now be opened directly from the storage location with the stored settings by a simple double-click.

5.7. Statistics

The dialogues of the Statistics area contain values which are related to the whole bitcontrol® Digital TV Link.

The window Overall (see Figure 5.17, “Window Overall - Statistics ” ) contains statistical values about global video and audio settings.

Figure 5.17. Window Overall - Statistics


The statistical values of the active Transcoders are shown in the Statistics window Transcoder.

Figure 5.18. Window Transcoder - Statistics


The statistical values in the third window are owned by the video stream of a channel of the selected plug-in. Figure 5.18, “Window Transcoder - Statistics ” shows the statistical values of the plug-in Dreambox as an example.

Note

The values of video jitter flood and video jitter famine can be used to detect the best Buffering Time for the plug-in.

Figure 5.19. Example for a plug-in window (Dreambox) - Statistics


You can switch between windows with the sliders in the headline.

5.9. Audio- and video decoder

The entries Audio Decoder and/or Video Decoder are visible if an audio and/or video decoder is/are used.

The configuration window - which is opened by clicking on one of the entries - depends on which audio and/or video decoder is used. Example: If bitcontrol® Video Decoder is used, the parametrization dialogue of bitcontrol® Video Decoder (see Figure 5.20, “Example for a parametrization window of a video decoder (bitcontrol® Video Decoder)”) opens.

Figure 5.20. Example for a parametrization window of a video decoder (bitcontrol® Video Decoder)


Note

The configuration windows for bitcontrol® Video Decoder in bitcontrol® Digital TV Link correspond to the bitcontrol® Video Decoder options dialogues. Further information about this topic can be found in the bitcontrol® Video Decoder user manual.
