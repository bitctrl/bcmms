21.2. Preferences

The window Preferences contains different possibilities to set RTSP Receiver properties.

Figure 21.3. Plug-in RTSP Receiver - Preferences


Authorization: RTSP Receiver user name and password can be set in the window area Authorization.

  • Name: The RTSP Receiver user name can be defined here. It can be necessary to set user name here if the URL doesn't contain a user name but coercively needs one for server authentication.
  • Password: The RTSP Receiver user password can be defined here. It can be necessary to set password here if the URL doesn't contain a password but coercively need one for server authentication.
  • Encryption level Authenticate "Basic" and Authenticate "Digest": The encryption level of the data transmission can be set by activating or deactivating the options Basic or Digest. With the authorization level Basic, the username and password are sent directly to an RTSP Server (e.g. Multistream Server, RTSP Media Session). Basic is a very simple but unsafe authorization method. With the authorization level Digest, the username and password are encrypted and sent to an RTSP Server (e.g. Multistream Server, RTSP Media Session). Digest is a complicated but safe authorization method.

The adjustable values in the RTP and the RTCP area represent socket parameters that can be used to optimize network data traffic. The following configurations are possible:

  • -1: The value range will be set automatically up to MTU size.
  • 0: The parameter will be not set.
  • KB value: Manual value range input in KB.

Transport: The window area Transport contains possibilities to set the type of data stream transport.

  • Query Transports: The data stream transmission methods unicast (default value), multicast or interleaved can be selected via this selection field. The list sequence shows the preferred methods (first place = "most prefered").
  • Query Multicast Address: Activating this option is only reasonable if the multicast transmission method was chosen in the Query Transports field. The entry of a destination server address (e.g. 329.0.0.1) is mandatory!

Send "OPTIONS": If this option is active, the client asks for server options (server methods and commands).

Ignore "Content-Base": If this option is active, the content (audio/video) will be interpreted in an alternative way. This option can be useful to solve conflicts on servers with more than one network interface card.

Ping with "GET_PARAMETER": If this option is active, the server will be pinged periodically to check its availability.

Average Video Timestamps: If this flag is set the timestamp will be corrected.

Note

This option should be used to improve the signal, when the video output stutters.

Accept "REDIRECT": If the option is active, the server sends a message that the content (video/audio) is on another server.

Cache-Addresses: This option activates the latching of network addresses.

AV Sync (lip-sync): This option activates the synchronisation of video and audio streams via RTCP.

Save RTP Dump File: If this option is active the RTP Dump File will be saved. The RTP Dump File is used for data stream analysis.

21.3. Links

The window Links shows a URL list of RTSP Servers.

Note

After a new installation the window is empty. The RTSP Server links must be created (see Section 21.3.1, “Create/add RTSP Server”).

21.3.1. Create/add RTSP Server

To create/add an RTSP Server the user clicks the right mouse button in the window Links. The submenu shown in Figure 21.4, “Create/add RTSP Server - Submenu” opens.

Figure 21.4. Create/add RTSP Server - Submenu


After selecting the entry Add link(s)... a new line will be created in the RTSP Server link list. The properties of the new RTSP Server can be edited by clicking on the respective table cell.

The new entry is indicated in red. After the confirmation with the Apply button, the entry will be coloured in black (see Figure 21.5, “Add RTSP Server”).

Figure 21.5. Add RTSP Server


21.3.2. Remove RTSP Server

To remove a RTSP Server out of the link list, the user marks the respective table line ands selects the entry Remove from the submenu (see Figure 21.4, “Create/add RTSP Server - Submenu”). The line of the RTSP Server will be removed from the URL list.

21.4. Statistics

The statistic information of the RTSP Receiver plug-in is self-explanatory in its designations.

The values are updated in real-time, so the RTSP Receiver information is always up to date. If the value of Renderer Lateness increases permanently, the hardware (CPU, graphic chip, bus system) will be overloaded. In this case it is very advisable to try the use of DXVA.
