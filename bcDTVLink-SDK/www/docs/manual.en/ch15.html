bitcontrol® Digital TV Link Capture plug-in allows playback, recording and/or streaming of video and/or audio streams from different capture sources.

Capture sources can, for example, be connected devices, like webcams or cameras, or the connected monitor (screencast).

If supported by the Capture source, it is possible to playback and record audio as well as control the devices (turn, tilt, zoom) and configure the video (brightness, contrast, sharpness, etc.).

Note

The available options for setting depend on the Capture source.

Note

It is best to use a separate Capture plug-in for each Capture source.

The plug-in can be started in the main window (see Figure 15.1, “Plug-in Capture - Main window”) with the help of the Start button and can be stopped with the Stop button.

Figure 15.1. Plug-in Capture - Main window


15.1. Other video Capture source requirements

The bitcontrol® Digital TV Link Capture plug-in uses all system available, compatible video/audio encoders and decoders by default. It is possible that the Capture source needs further video/audio encoders and decoders to work correctly and/or to get a better playback quality. The required video/audio encoders and decoders have to be installed individually or as package components (e.g. in ffdshow) and configured.

Configuration example for ffdshow

To get a better video quality with the webcam, which is used here as an example Capture source, ffdshow was installed as well and configured as in Figure 15.2, “EXAMPLE - ffdshow configuration” (Encoder: H.264, FourCC: H264, Modus: one pass - average bitrate).

Note

ffdshow must be configured before being used. The ffdshow configuration window can be opened via the start menu Start/All Programs/ffdshow/VFW-Configuration.

Figure 15.2. EXAMPLE - ffdshow configuration


The selection of ffdshow as the codec for the Capture source occurs directly via Compressor table field of the Options window of Capture plug-in (see Section 15.2, “Options”).

15.2. Options

All available Capture devices are shown in the table of the Options window (see Figure 15.3, “EXAMPLE Capture device with ffdshow as codec”) of the Capture plug-in.

Figure 15.3. EXAMPLE Capture device with ffdshow as codec


The following properties for each Capture device can be parameterized in the respective table column:

Type: This field contains the type of Capture data stream. The choices are: Audio and Video.

Capture Source: This selection field is used to set the connected Capture device (e.g. webcam) as the data stream source.

Compressor: The codec to be used for playing back the Capture data stream can be chosen via the selection field. All codes installed on the system can be chosen (see Figure 15.4, “Codec selection field for a Capture device”).

Figure 15.4. Codec selection field for a Capture device


Capture Device ID: ID of the Capture device.

Fields are edited via double-click and selection field and/or by hand.

The functionalities of the buttons Add, Remove and Configure correspond to the Capture plug-in context menu options (see next part).

Information about the Capture source can be added in the editable fields Name and Description. This information will be visible in the Channels list (see Chapter 6, Channels and Figure 15.6, “Capture source information in Channels list”) when the Capture plug-in runs and the respective Capture source was selected.

Figure 15.5. Capture source information in the lower part of the Capture plug-in window


Figure 15.6. Capture source information in Channels list


Capture plug-in context menu

Right clicking the Options window of the Capture plug-in opens the context menu (see Figure 15.7, “Capture plug-in context menu”). It is used to add, remove or configure the Capture device.

Figure 15.7. Capture plug-in context menu


15.3. Add Capture source

To add a Capture source to the list, the user selects the entry Add from the context menu (see Figure 15.7, “Capture plug-in context menu”). A new row will be added to the table (see Figure 15.8, “Add a Capture device”). The new Capture source can now be edited or configured (see Section 15.5, “Configure Capture source”).

Figure 15.8. Add a Capture device


Note

Depending on the selected Type, all available Capture sources will be shown in the Capture Source selection field that support this Type. There should be a reasonable combination of Capture source devices per Capture plug-in.

For example - video and audio from the same room: The camera, which is situated in the same room as the selected microphone, should be selected as the video source out of all proposed camera devices.

15.4. Remove Capture source

To add a Capture source to the list the user selects the entry Add from the context menu (see Figure 15.7, “Capture plug-in context menu”). The marked line of the Capture source disppears from the table.

15.5. Configure Capture source

If the entry Configure was selected from the Capture context menu (see Figure 15.7, “Capture plug-in context menu”), further dialogues open. These are used to set more Capture source properties.

Note

The available wizards depend on the type of Capture source.

15.5.1.2. ProcAmp

The window ProcAmp contains setting possibilities for displaying the videos in bitcontrol® Digital TV Link.

Figure 15.11. Plugin Capture - ProcAmp


Brightness: The brightness of the video can be adjusted with this slider.

Contrast: The contrast of the video can be adjusted with this slider.

Hue: The hue of the video can be adjusted with this slider.

Saturation: The saturation of the video can be adjusted with this slider.

Sharpness: The sharpness of the video can be adjusted with this slider.

Gamma: The gamma value of the video can be adjusted with this slider.

White Balance: The white balance of the video can be adjusted with this slider.

Gain: The gain of the video can be adjusted by this slider.

Color Enable: The color (grey to colour) of the video can be adjusted with this slider.

Blacklight The blacklight of the video can be adjusted with this slider.

Reset: The default settings can be restored with the Reset button.

Note

Unavailable functions are greyed out!

15.5.2. Screencast - Screen as Capture source

15.5.2.1. Screen window

Figure 15.13, “Configuration area of the Screen Capture source” shows the Screen window of the configuration area of the Screen Capture source.

Figure 15.13. Configuration area of the Screen Capture source


This window is used to set the properties for the screencast functionality of bitcontrol® Digital TV Link.

Input

Position: The position fields contain the position (left X Position and right Y Position) of the screen area that should be captured.

Size: The position fields contain the size (left Width and right Height) of the screen area that should be captured.

FPS: The frame rate that should be used for screen capturing can be selected here.

Capture Mouse: If this checkbox is active, mouse activities are captured too.

Capture Layered Windows: If this checkbox is active, half-transparent Windows pop-up windows will be deactivated.

Disable Window Aero: If this checkbox is active, the Windows Aero user interface will be deactivated. Deactivating this option may enlarge performance.

Linear Upscale Filter: If this checkbox is active, the screen area that should be captured will be upscaled with a linear filter.

Output

Size: This field contains the size (left Width and right Height) of the captured screen video.

Format: This field contains the format of the captured screen video.

Rotate: By selecting one of the provided entries, the captured screen video will be played back turned by 90 degrees, 180 degrees or 270 degrees.

Flip Vertical: If this checkbox is active, the captured screen video will be played back vertically rotated.

Flip Horizontal: If this checkbox is active, the captured screen video will be played back horizontally rotated.

YUV Format

In this window area the following settings can be parametrized for color conversion to YUV:

Matrix Coefficients: This selection field will be used to set the color conversion method. In addition to standard methods, the entry Custom is provided too. If this entry is selected, the values for Cr, Cg and Cb can be edited. The sum of all values should not exceed the value 1.0.

Full Range YUV: The activation of this checkbox is reasonable when the default settings show wrong colors.

Note

Full Range must be supported by the hardware being used.
