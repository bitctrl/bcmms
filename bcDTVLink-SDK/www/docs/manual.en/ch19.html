19.2.1. Video recording via scheduler

Video streams of different sources can be recorded with the help of the plug-in Recorder. Furthermore it is possible to parameterize the records here.

19.1. Options

General configurations for video recording, such as the storage location, file format or video size, can be adjusted via the Options window (see Figure 19.1, “Plug-in Recorder”).

Figure 19.1. Plug-in Recorder


Filename: The input field specifies the place (the directory) and the name of the file to save. The first part of the string should denote the directory in which to save in, e.g. C:\TEMP\.

Further on the name of the file has to be stated. The following wild cards are accepted:

Note

The addition of a format extension isn't necessary here.
$CHANNEL$Channel name (default value)
$PROVIDER$Provider name
$NR$Consecutive number in the directory
$DESKTOP$Windows desktop
$DESKTOPDIRECTORY$User desktop
$MYPICTURES$IE 5.0 – My picture folder
$PERSONAL$My data folder
%aAbbreviated weekday name
%AFull weekday name
%bAbbreviated month name
%BFull month name
%cDate and time representation
%CCentury (00 - 99)
%dDay of month as decimal number (01 – 31)
%DDate(%m %d %y)
%eDay of month
%gYear without century
%GYear with century
%HHour in 24-hour format (00 – 23)
%IHour in 12-hour format (01 – 12)
%jDay of year as decimal number (001 – 366)
%mMonth as decimal number (01 – 12)
%MMinute as decimal number (00 – 59)
%pCurrent locale's A.M./P.M. indicator for 12-hour clock
%ra.m. or p.m.
%RTime in 24 hours format
%SSecond as decimal number (00 – 59)
%TCurrent time (%H:%M:%S)
%uDay of the week (1(Mo) - 7(So))
%UWeek of year as decimal number, with Sunday as first day (00 – 53)
%wWeekday as decimal number (0 – 6; Sunday is 0)
%WDay of the week; 0 (sun) to 6 (sat)
%xWeek of year as decimal number, with Monday as first day (00 – 53)
%XDate
%yTime
%YYear as 2-digit value
%zYear as 3-digit value
%ZTime-zone name or abbrev, no characters if time zone is unknown
%%Percent sign

The button Browse next to the text field opens a Windows typical dialogue, which can be used to select the storage path of the video data.

Save Elementary Streams: If this option is active, the elementary data streams (audio, video,...) will be stored too.

File Types: The video file format can be set here in the form of a format extension. It is possible to name more than one format. In this case the first format will be used preferentially. If it isn't possible, the next format follows and so on.

Splitting Limits: Here you can set the limits in volume (in MBytes) and duration (in minutes).

The user can choose between the following possibilities:

  • Volume (MByte): unlimited, 600, 800, 1000 and 4000
  • Duration (Min): unlimited, 60, 120 and 240

Enable Autodelete: If this check box is active, recordings that are older than the defined value in text field Lifetime (min) will be deleted. The check box isn't active by default.

Note

The plug-in area Autodelete is important for permanent recording.

Note

You will find a manual for recording video files (e.g. a TV broadcast) in Section 19.2.1, “Video recording via scheduler”.

19.2. Scheduler

The window Scheduler (see Figure 19.2, “Selection of the entry Add entry”) contains a list of all video streams that are marked for recording.

The following information are visible in the table:

  • Date (yyyy:mm:dd): Recording date in format year:month:day
  • Time (hh:mm:ss): Recording start time in format hour:minute:second
  • Duration (hh:mm:ss): Recording duration in format hour:minute:second
  • Receiver: Video stream input source (e.g. Dreambox)
  • Channel: Channel that transports the video stream (depend on input source)
  • Priority: Possibility to configure a priority for overlapping records (possible are: High, Normal and Low.)
  • Description: Video stream description

19.2.1. Video recording via scheduler

An video stream can be marked manually for recording in the window Scheduler. To do so, follow these steps:

Clicking the right mouse button opens the sub menu shown in Figure 19.2, “Selection of the entry Add entry”. After selecting the entry Add entry, a new entry appears in the table. The parameters of the entry correspond to the parameters of the selected input source and can be adjusted separately here (see Figure 19.3, “Edit parameters”). The settings will be assumed afterwards with the Apply button. The record is prepared now.
