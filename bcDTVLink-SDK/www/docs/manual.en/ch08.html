8.1. Decoder

A decoder transforms a specific audio or video format into an audio or video stream.

After clicking on the entry Decoder in the tree in the left part of the Options dialogue, the table shown in Figure 8.1, “Overview over all installed and created audio and video decoder” appears. The table contains all installed and created audio and video decoders. The different decoder types can be distinguished by their different background colors: pink (audio), yellow (video) and blue (common).

Figure 8.1. Overview over all installed and created audio and video decoder


The following decoder properties can be taken from the table:

Name: Description of the decoder

Type: The decoder type can be set in this field (only created decoders). the choices are: Audio, video and common. If the entry common is selected, the decoder can be an audio and video decoder.

System: This field shows if the decoder is installed on the system (yes) or was created by the user (no). The field cannot be edited.

Topology: This column shows, with yes or no whether the decoder owns a Topology (for further information see Section 8.3.1, “Topology”) or not. Installed decoders have a prefabricated Topology. They cannot be edited.

Properties: This column shows, with yes or no whether the decoder properties were changed.

DeviceID: This column contains the DeviceID of the decoder.

Attributes: The following attributes can be selected for the decoder Set: ForceZeroOutputTimestamp or Set: NoThreadDirectShowFilterGraph. The selection occurs via the selection field (see Figure 8.2, “Decoder Attribute selection field”), which opens is opened by double clicking on the Attribute field of the decoder.

Figure 8.2. Decoder Attribute selection field


Set: ForceZeroOutputTimestamp: The decoder (or encoder) could have timestamp bugs. If this property is selected, the timestamp will be newly calculated. The timestamp of the decoder (or encoder) will not be used.

Set: NoThreadDirectShowFilterGraph: If this attribute was selected, the decoder doesn't start an extra thread, so the speed of video playback can be optimized.

8.1.1. Add decoder

Right clicking in the window opens the decoder context menu (see Figure 8.3, “Decoder context menu”) and the entry Add will be selected there.

Figure 8.3. Decoder context menu


A new row (see Figure 8.3, “Decoder context menu”) will be added to the table. Now the user can parameterize and configure the decoder however he wants.

8.1.2. Decoder parametrization / configuration

The functions for decoder parametrization / configuration are available for created decoders and copies of installed decoders via the decoder context menu (see Figure 8.3, “Decoder context menu”). The context menu can be opened with a right click and it provides the following possibilities:

  • Add: A new decoder can be added with this entry (see Section 8.1.1, “Add decoder”).

  • Remove: A decoder can be removed from the table with this entry.

  • Clone: A copy of the selected decoder will be added to the table with this entry.

  • Edit Topology: This entry opens the Topology editor. It is empty if the decoder was just created, so the user is able to write the Topology of the decoder by hand. Otherwise, the window shows an XML structure that the user can edit.

  • Create Default Topology: This entry allocates a prefabricated Topology to the decoder. This Topology can be edited by the user.

  • Delete Topology: This entry will be used to delete the Topology.

  • Configure: This entry opens the configuration window of the decoder. The adjustment possibilities are provided by the decoder producer. For an example see Figure 8.4, “Decoder configuration window of a third-party producer”.

Figure 8.4. Decoder configuration window of a third-party producer


Note

Some fields of this decoder configuration window can be edited through selection fields or by hand. The changing of preset values should only be done by trained users!

  • Edit Properties: If this entry is selected, a window (see Figure 8.5, “Decoder configuration editor”) opens where the decoder properties can be edited by hand.

    Note

    The context menu entry Edit Properties is available when the decoder properties were changed in the configuration window (see Figure 8.4, “Decoder configuration window of a third-party producer”).

Figure 8.5. Decoder configuration editor


  • Delete Properties: If this entry is selected, the default settings of the decoder will be restored. The table entry changes from yes to no.

    Note

    The context menu entry Delete Properties is available when the decoder properties were changed in configuration window (see Figure 8.4, “Decoder configuration window of a third-party producer”).

8.2. Encoder

A encoder transforms the signal data of a source (e.g. an audio or video stream) into an adequate format.

The windows, menus and parametrization possibilities correspond to Decoders (see Section 8.1, “Decoder” ).

8.3. Transcoder

A transcoder will be used to transform an input data stream (audio and/or video) of any format into an output data stream (audio and/or video) of any format.

Note

Transcoding video streams requires a lot of processing power !

The table Transcoders (see Figure 8.6, “Transcoders table”) in the window, which is opened by double clicking on the tree entry Transcoders in the left area of Options window, shows all configured transcoders and their properties.

Figure 8.6. Transcoders table


Name: This field contains the name of the transcoder.

Type: This selection field is used to choose the input stream type (video, audio or other data stream). The following types are selectable: common, video and audio.

Topology: This field shows with yes or no whether there is a Topology (see Section 8.3.1, “Topology”) for the transcoder.

Decoder: This field shows the name of the decoder being used for the transcoder.

Encoder: This field shows the name of the encoder being used for the transcoder.

Attributes: The properties Set:ForceZeroOutputTimestamp and Set:NoThreadDirectShowFilterGraph can be assigned to the transcoder here via a selection field (see Figure 8.2, “Decoder Attribute selection field”). For more information, see Section 8.1, “Decoder”.

8.3.1. Topology

A Topology (see Figure 8.7, “Topology editor”) is an XML structure in which contains:

  • ... the used decoder (line 3)
  • ... the used encoder (line 4)
  • ... the decoder input pin that the data source is connected to (line 7)
  • ... which decoder output pin is connected to which encoder input pin (line 8)
  • ... the encoder outpit pin that the data sink is connected to (line 9)

Figure 8.7. Topology editor


For pin configuration counts: 0 equates to the first pin, 1 equates to the second pin, 2 equates to the third pin and so on.

Note

The XML trancoder configuration is necessary and very individual. It must be customized to the respective system.
